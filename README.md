# 企业库 / 产品库 / 备份管理系统（Flask）

本系统是一个面向资料沉淀与归档的轻量化后台，**当前仅保留三大模块**：

1. 企业库
2. 产品库
3. 备份管理

系统已清理并隐藏外资客户、需求匹配、项目进展、复杂看板、撮合评分等非核心入口，界面导航只保留上述三个模块。

---

## 系统简介

- 技术栈：Flask + SQLAlchemy + SQLite + Jinja2 + Bootstrap
- 主要目标：
  - 企业资料的结构化录入、查询、维护与归档
  - 产品资料的结构化录入、查询、维护与归档
  - 数据库与文件资料的可下载备份、可恢复

默认登录账号：

- 管理员：`admin / admin123`
- 普通用户：`user / user123`

> 生产环境请务必立即修改默认账号密码。

---

## 三大模块说明

### 1) 企业库

支持能力：

- 新增企业
- 编辑企业
- 查看企业详情
- 查询筛选（名称、省市、行业、企业性质、出口经验、销售额区间等）
- 行业分类（行业代码 + 行业名称）
- 行业专项字段（按行业动态展示）
- 文件上传（企业级资料）
- 企业文件夹归档（自动归入企业专属目录）
- Excel 导入导出

### 2) 产品库

支持能力：

- 新增产品
- 编辑产品
- 查看产品详情
- 查询筛选（关键词、所属企业、行业、HS、MOQ、周期等）
- 行业分类（继承或手动选择）
- 行业专项字段（按行业动态展示）
- 文件上传（产品资料）
- 产品文件归档（归档到所属企业目录下的对应分类）
- Excel 导入导出

### 3) 备份管理

支持能力：

- 数据库备份
- 企业文件资料备份
- 完整备份（数据库 + 文件）
- 下载备份文件
- 每日启动自动备份数据库（当天首次启动触发一次）

## 行业分类说明

- 行业分类由 `config/industry_config.py` 统一维护。
- 企业表单行业字段：`industry_code` / `industry_category`。
- 产品表单行业字段：`industry_code` / `industry_name`。
- 行业专项字段配置：
  - 企业：`config/enterprise_form_config.py`
  - 产品：`config/product_form_config.py`

系统会按所选行业动态渲染专项字段，未配置的行业不会展示专项分组。

## 企业录入页改造说明（顶部固定区 + A-J 模块）

企业新增/编辑页采用统一表单模板（`templates/enterprise_form.html`）：

- 顶部固定概览区：展示企业编号、状态、行业、项目负责人、资料完整度、建议入库状态。
- 折叠模块结构：
  - 入库信息（固定区）
  - A-I：企业通用模块（由 `config/enterprise_form_config.py` 驱动）
  - 行业专项模块：按行业代码动态插入（由 `config/industry_config.py` 驱动）
  - J：附件资料模块（支持上传、下载、删除）
- 模块完成状态：每个折叠模块会显示“已完成/待完善（x/y）”。
- 联动显示：支持 `show_when` / `show_when_any` 条件展示（如是否自有工厂、是否已有海外业务、是否有出口经验、是否上市等）。
- 资料完整度：按“最小必填 + 建议字段 + 附件”综合计算，并在新增/编辑/列表/详情页展示。

## 文件归档规则

上传根目录：`uploads/企业库/`

归档路径规则：

- 一级：企业专属目录（基于企业编号与企业名称）
- 二级：标准分类目录（如企业基础资料、认证报告、报价资料、图片视频等）
- 文件名：系统自动清洗特殊字符、按规则生成并避免重名覆盖

补充规则：

- 禁止扩展名：`.exe/.bat/.js/.sh/.cmd/.com/.msi/.ps1`
- 单文件大小限制：100MB
- 上传记录会写入数据库 `documents` 表，支持列表筛选和下载

## 备份与恢复说明

### 备份目录

- 数据库备份：`backups/database/`
- 文件备份：`backups/files/`

### 备份类型

- 数据库备份：`database_backup_YYYYMMDD_HHMMSS.sqlite`
- 文件资料备份：`enterprise_files_backup_YYYYMMDD_HHMMSS.zip`
- 完整备份：同一操作内同时生成上述两类文件

### 自动备份

- 应用启动时会检查当天数据库备份是否已存在。
- 若不存在，当天首次启动自动创建一份数据库备份。

### 恢复建议流程

1. 停止服务
2. 备份当前数据库与 uploads 目录
3. 用目标备份覆盖数据库文件（`trade_agent.db`）
4. 如需恢复附件，解压 `enterprise_files_backup_*.zip` 到项目根目录
5. 重启服务并验证关键页面

## 本地运行方式

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动系统

```bash
python app.py
```

启动后访问：

- 登录页：`http://127.0.0.1:5000/登录`
- 企业库：`http://127.0.0.1:5000/enterprises`
- 产品库：`http://127.0.0.1:5000/products`
- 备份管理：`http://127.0.0.1:5000/backup`

## 数据兼容与迁移说明

为避免历史数据报错，系统启动初始化时会自动执行兼容检查：

- 若缺少企业扩展字段列：自动补齐 `enterprise_extra_fields`
- 若缺少产品行业字段：自动补齐 `industry_code` / `industry_name`
- 若缺少产品扩展字段：自动补齐 `product_extra_fields`
- 若缺少产品状态字段：自动补齐 `status` 并回填默认值

同时会执行一次产品行业字段回填（从所属企业继承），以保证旧数据可读。
